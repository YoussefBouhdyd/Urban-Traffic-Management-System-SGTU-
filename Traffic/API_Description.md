# REST API Specification for Dashboard



# 1-  Get latest traffic state

## Endpoint

```http
GET /api/traffic/latest
```

## Purpose

Returns the **latest analyzed traffic state** for the intersection.

This endpoint is mainly for:

* real-time dashboard card
* current traffic status
* latest recommendation
* current severity level

## Query parameters

None

## Success response

**Status:** `200 OK`

```json
{
  "intersectionId": "INT-01",
  "timestamp": "2026-03-15T17:25:00",
  "trafficState": "CONGESTED",
  "severity": "HIGH",
  "recommendation": "Extend green light for north-south road",
}
```

## Response fields

* `intersectionId`: unique intersection id
* `timestamp`: time of latest analysis
* `trafficState`: current traffic state
  Possible values:

  * `NORMAL`
  * `BUSY`
  * `CONGESTED`
  * `INCIDENT`
* `severity`: seriousness level
  Possible values:

  * `LOW`
  * `MEDIUM`
  * `HIGH`
  * `CRITICAL`
* `recommendation`: action proposed by the system

# 2- Get traffic history

## Endpoint

```http
GET /api/traffic/history
```

## Purpose

Returns the **historical analyzed traffic events**.

This endpoint is mainly for:

* charts
* timeline/history table
* filtering by date
* showing traffic evolution over time

## Query parameters


* `from` : start datetime in ISO 8601
* `to` : end datetime in ISO 8601

## Example request

```http
GET /api/traffic/history?from=2026-03-15T08:00:00&to=2026-03-15T18:00:00
```

## Success response

**Status:** `200 OK`

    "intersectionId": "INT-01",
    "timestamp": "2026-03-15T09:00:00",
    "trafficState": "NORMAL",
    "severity": "LOW",
    "vehicleCount": 12,
    "averageSpeed": 38.5,
    "recommendation": "No action needed"
  },
  {
    "id": 102,
    "intersectionId": "INT-01",
    "timestamp": "2026-03-15T12:30:00",
    "trafficState": "BUSY",
    "severity": "MEDIUM",
    "vehicleCount": 29,
    "averageSpeed": 24.1,
    "recommendation": "Monitor traffic density"
  },
  {
    "id": 103,
    "intersectionId": "INT-01",

# 3- Get active alerts

## Endpoint

```http
GET /api/alerts
```

## Purpose

Returns the list of **current or recent alerts** generated by the central analysis system.

This endpoint is mainly for:

* alert panel
* notifications area
* red warning cards
* urgent dashboard events

The source explicitly requires the system to **generate alerts** and support traffic recommendations. 

## Query parameters

None

## Success response

**Status:** `200 OK`

```json
[
  {
    "id": 1,
    "type": "CONGESTION",
    "severity": "HIGH",
    "message": "Heavy congestion detected at intersection INT-01",
    "timestamp": "2026-03-15T17:25:00",
    "status": "ACTIVE"
  },
  {
    "id": 2,
    "type": "INCIDENT",
    "severity": "MEDIUM",
    "message": "Possible incident detected near pedestrian crossing",
    "timestamp": "2026-03-15T16:55:00",
    "status": "ACTIVE"
  }
]
```

## Response fields

* `id`: alert id
* `type`: alert category
  Possible values:

  * `CONGESTION`
  * `INCIDENT`
  * `CAMERA_OFFLINE`
  * `SYSTEM_WARNING`
* `severity`: alert level
* `message`: human-readable alert text
* `timestamp`: alert creation time
* `status`: state of the alert
  Possible values:

  * `ACTIVE`
  * `RESOLVED`

## Notes for frontend

* If empty array is returned, display “No active alerts”
* This endpoint can be polled every few seconds for near real-time updates

## Empty response example

**Status:** `200 OK`

```json
[]
```

# 4- Get recommendations

## Endpoint

```http
GET /api/recommendations
```

## Purpose

Returns the latest **system-generated recommendations** for the intersection.

This endpoint is mainly for:

* recommendation widget
* operations panel
* decision support display

The project source explicitly asks for recommendations such as adjusting traffic lights, opening another lane, or diverting traffic. 

## Query parameters

None

## Success response

**Status:** `200 OK`

```json
[
  {
    "id": 11,
    "intersectionId": "INT-01",
    "timestamp": "2026-03-15T17:25:00",
    "recommendation": "Increase green light duration by 20 seconds",
    "reason": "Vehicle density is high and average speed is low",
    "priority": "HIGH"
  },
  {
    "id": 12,
    "intersectionId": "INT-01",
    "timestamp": "2026-03-15T17:26:30",
    "recommendation": "Monitor pedestrian crossing zone closely",
    "priority": "MEDIUM"
  }
]
```

## Response fields

* `id`: recommendation id
* `intersectionId`: related intersection
* `timestamp`: creation time
* `recommendation`: recommended action
* `priority`: importance of the recommendation
  Possible values:

  * `LOW`
  * `MEDIUM`
  * `HIGH`

## Empty response example

**Status:** `200 OK`

```json
[]
```

# 5- Get camera status

## Endpoint

```http
GET /api/cameras/{id}/status
```

## Purpose

Returns the current status of a camera simulator.

This endpoint is mainly for:

* camera health/status card
* backend monitoring
* showing whether simulator is running or not

## Path parameters

* `id`: camera id

## Example request

```http
GET /api/cameras/CAM-01/status
```

## Success response

**Status:** `200 OK`

```json
{
  "cameraId": "CAM-01",
  "intersectionId": "INT-01",
  "status": "RUNNING",
  "lastUpdate": "2026-03-15T17:25:00"
}
```

## Response fields

* `cameraId`: unique camera id
* `intersectionId`: linked intersection
* `status`: current simulator status
  Possible values:

  * `RUNNING`
  * `STOPPED`
  * `ERROR`
* `lastUpdate`: last received event timestamp
