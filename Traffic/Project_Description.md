# Main business logic

## Camera event generation

The camera simulator generates one metadata event every few seconds, for example every 5 seconds.

Values can be randomized within realistic ranges:

* `vehicleCount`: 0 to 60
* `averageSpeed`: 0 to 60
* `stoppedVehicles`: 0 to 15
* `pedestrianCrossing`: true/false
* `suspectedIncident`: true/false
* `congestionHint`: LOW / MEDIUM / HIGH

## Traffic analysis rules

Use simple deterministic rules like:

### Rule 1: Normal

If:

* `vehicleCount < 20`
* `averageSpeed >= 30`
  Then:
* `trafficState = NORMAL`
* `severity = LOW`
* `recommendation = "No action needed"`

### Rule 2: Busy

If:

* `vehicleCount between 20 and 35`
* `averageSpeed between 15 and 30`
  Then:
* `trafficState = BUSY`
* `severity = MEDIUM`
* `recommendation = "Monitor traffic density"`

### Rule 3: Congested

If:

* `vehicleCount > 35`
* `averageSpeed < 15`
  Then:
* `trafficState = CONGESTED`
* `severity = HIGH`
* `recommendation = "Increase green light duration by 20 seconds"`

### Rule 4: Heavy congestion

If:

* `vehicleCount > 45`
* `averageSpeed < 8`
* `stoppedVehicles > 6`
  Then:
* `trafficState = CONGESTED`
* `severity = CRITICAL`
* `recommendation = "Apply traffic diversion strategy"`

### Rule 5: Incident

If:

* `suspectedIncident = true`
  Then:
* `trafficState = INCIDENT`
* `severity = HIGH` or `CRITICAL`
* `recommendation = "Dispatch verification and consider route deviation"`

### Rule 6: Pedestrian warning

If:

* `pedestrianCrossing = true`
* and `vehicleCount` is high
  Then:
* create a medium-level alert
* add recommendation like `"Monitor pedestrian crossing zone closely"`
